Admin-Ajax API

The plugin does not register public REST routes for testing. All server interaction from the admin UI uses POST to admin-ajax.php with a WordPress AJAX action. Responses use the standard wp_send_json_success / wp_send_json_error envelope.

Localized script: wprrt_vars

Injected via wp_localize_script on the admin screen handle wprrt-script:

KeyDescription
ajax_urlURL to admin-ajax.php
nonceNonce string for action wprrt_nonce (POST field name nonce)

Actions

Send POST to ajax_url with action set to one of the values below, plus nonce.

wprrt_get_routes

Returns all registered REST routes with methods. Nonce required; no extra capability beyond logged-in admin context for the AJAX call.

Success data: object keyed by route path; each value includes methods (array) and primary_method (string).

wprrt_get_user_roles

Returns role slugs and labels for the role dropdown, plus an empty string key for “Default (Current User)”.

Success data: object { [roleSlug: string]: string }

wprrt_test_route

Executes rest_do_request() for the given route. Requires current_user_can( 'manage_options' ).

POST fieldRequiredNotes
nonceyesMust verify against wprrt_nonce
routeyesREST path (e.g. /wp/v2/posts)
methodnoDefault GET. Allowed: GET, POST, PUT, DELETE
headersnoJSON object string (parsed server-side)
formnoJSON object string
bodynoJSON string; parsed for POST/PUT body params
rolenoIf set, a temporary user is created with that role for the request, then deleted

Success data: REST response body from WP_REST_Response::get_data() (shape depends on the route).

Errors: invalid method, invalid JSON, insufficient permissions, user creation failure, etc.—returned via JSON error envelope.

Example (conceptual)

action=wprrt_get_routes&nonce=...

Do not expose the nonce to untrusted browsers; this reference is for developers extending the admin UI only.
